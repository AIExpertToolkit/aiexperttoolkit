Exactly what you’ll get

  • Production Jekyll site on GitHub Pages (custom domain, DNS, CNAME).
  • SEO hygiene: single canonical, robots, sitemaps; Organization/WebSite/Article/Breadcrumb structured data.
  • Core Web Vitals pass targets on home + one money page (Perf & AX ≥ 95 in Lighthouse, CLS < 0.02).*
  • Accessibility (WCAG 2.1 AA): skip links, landmarks, focus states, color contrast, alt text audit.
  • Monetization readiness: affiliate link hygiene (rel, UTM), GA4 event logging (click_affiliate, click_nav, click_consult), CRO CTAs.
  • Content scaffolds: comparison, review, pricing templates; breadcrumbs; ToC; FAQ schema.
  • Proof-of-build: daily commits, short Looms, and written SOPs (deploy/validate/new page).

* Targets depend on host/theme constraints; we surface trade-offs and track results in a validation log.

Scope & out-of-scope (so expectations are clear)

Included

  • Jekyll layouts/includes refactor where needed (GH Pages-safe).
  • Header/footer hardening, breadcrumbs, ToC, FAQ blocks.
  • GA4 + GSC setup/verification (if not already present).
  • Image hygiene (sizes/decoding/lazy; modern formats when practical).
  • Light client-side search (if feasible without external services).

Not included

  • Custom backends, server-side apps, or databases.
  • Heavy redesigns/brand systems or multi-language sites.
  • Large content migrations (>20 pages) in the sprint window.
  • Paid ads or complex analytics pipelines.

How the 10-day sprint runs

  1. Day 1: Kickoff, repo audit, DNS/Pages, baseline Lighthouse/AX.
  2. Days 2–3: Layout/header/CWV hardening, GA4/GSC, robots/sitemap.
  3. Days 4–6: Bottom-funnel pages (review, vs, pricing) + CRO CTAs.
  4. Days 7–8: E-E-A-T surfaces (About, Methodology, Editorial Policy), internal linking.
  5. Day 9: Validation runbook, Rich Results, URL Inspection, fixes.
  6. Day 10: Handover: SOPs, recordings, roadmap, Q&A.
